Overview
Built for recruiters who think like operators.

Gibbons is a recruiting intelligence plugin built on Claude AI. It runs as a structured, step-by-step workflow — from the moment a hiring request lands to the moment a candidate receives a personalized outreach message.

Every output is grounded in company context, hiring manager signals, and verifiable candidate evidence. There is no generic output. If Gibbons can't cite it, it doesn't say it.

The system operates in two phases: a role intake phase that builds the job, the interview process, and the success definition — and a calibration and outreach phase that validates candidates and crafts outreach that creates real friction and real responses.

Context carries forward between sessions via structured handoff blocks, so no recruiter has to re-paste the same job description three times to get the work done.

Candidate Truth Rules

Every claim about a candidate must trace to explicit evidence in their profile. No inference. No gap-filling. If it's not stated, it's "not stated."

Section B + C Context Binding

All output is shaped by company values, ideal candidate profile, and recruiter voice. If the output could apply to any company, it fails.

Carry Forward Architecture

Each skill ends with a structured handoff block. Paste it into the next session. Zero re-work. Full context preserved.

Memory Persistence

Company research and kickoff notes are stored per-role in a memory directory so any session can pick up exactly where the last left off.

Design Principles
The rules that make every output trustworthy.

Gibbons enforces a layered constraint system across every skill. These aren't suggestions — they're hard checks built into each workflow.

🔍

Candidate Truth Rules

Every claim about a candidate must trace to explicit text in their profile. No inferences. No pattern-matching. If information is missing, the output says "not stated" — not a plausible guess.

🎯

Context Binding

Section B (voice and tone) and Section C (ICP and values) are authoritative context that shapes every output. If the deliverable could apply to any company, it fails the check.

⚖️

Scope Constraints

Do exactly what was asked. No strategic additions, no unsolicited recommendations, no broadened mandates. Adjacent ideas go in an "Optional extras" section — max 3 bullets, never expanded.

🛑

Non-Negotiables

A hard list of phrases, patterns, and behaviors that never appear in any output — regardless of context. These override all stylistic defaults and apply to every word written on the recruiter's behalf.

📋

Assumption Discipline

When inputs are missing, Gibbons chooses a practical default and labels it clearly under "Assumptions." Assumptions are never blended into core output or presented as facts.

✅

Verification Gates

Before final output, each skill runs a compliance check. Outreach messages run a source map, a Section B absorption check, and a could-you-send-this-to-ten-people test. Failures trigger a rewrite before anything is shown.
